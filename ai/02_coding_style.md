# 代码规范

## 基本要求
- 使用 Python 3.13 语法
- 使用 type hints
- 公共函数必须有 docstring
- 单个函数尽量只做一件事
- 复杂解析逻辑必须拆成独立函数

## 命名规范
- 变量、函数：snake_case
- 类名：PascalCase
- 常量：UPPER_CASE
- 输出文件名应表达站点与数据类型，如 majors-flat.json、all.json、meta.json

## 日志规范
- 使用 logging，不使用零散 print 作为正式日志
- 日志至少包含：阶段、筛选条件、URL、异常信息
- 对于长流程任务，必须输出进度日志

## 结构规范
- 页面访问、字段提取、详情解析、文件写入必须分离
- 列表页解析函数不能直接承担详情页结构化逻辑
- 所有写文件逻辑统一封装

## 异常处理
- 必须捕获页面加载异常、选择器失效、字段缺失、写文件异常
- 异常发生时保留 partial 输出，避免整批数据丢失
