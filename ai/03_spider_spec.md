# 爬虫规范

每个爬虫必须定义：
- base_url
- headers
- 请求频率
- 解析规则

## 请求规则
- 随机 UA
- timeout
- retry

## 并发
- 最大并发：5
- 间隔：1s

---

## CHSI 专业采集要求

### Stage 1：列表页
必须提取：
- 培养层次
- 门类
- 专业类
- 专业名称
- 专业代码
- specId
- 详情页
- 专业满意度
- 列表来源页

规则：
- 列表页只负责基础字段
- 所有链接转绝对 URL
- 优先按 specId 去重
- 必须记录筛选上下文与分页信息

### Stage 2：详情页
必须至少解析：
- 顶部信息
- basic_info
- schools
- courses
- raw

规则：
- 基本信息、院校页、课程页分函数处理
- 能结构化的列表与表格必须结构化
- 允许局部失败，不影响整条专业记录输出，但要记录失败位置
- 必须保留来源 URL 与抓取时间

## CHSI 院校采集要求

### Stage 1：列表页
必须提取：
- 学校名称
- schId
- 详情页
- 学校图片
- 主管部门
- 院校所在地
- 办学层次
- 学校类型
- 院校满意度
- 列表来源页
- 页码

规则：
- 优先从详情链接提取 schId
- 支持分页
- 列表图片必须采集，但不作为主键
- 字段缺失时允许留空，不得编造

### Stage 2：详情页
当前至少解析：
- 顶部信息（home/header）
- intro

规则：
- 每个页面单独函数
- 允许仅保存结构化摘要 + raw_text
- 联系方式、链接、地址等优先结构化
- 页面缺失时返回空对象，不报整条失败，但要记录失败位置

## XZ 专业采集要求

### Stage 1：列表页
必须提取：
- 专业名称
- specId
- 详情页
- 列表来源页
- 页码

规则：
- 列表页只负责基础字段与详情入口
- 所有链接转绝对 URL
- 优先按 specId 去重
- 必须保留筛选条件与分页上下文

### Stage 2：详情页
必须按标签页解析：
- 顶部信息
- basic_info
- universities
- courses
- graduated_development
- raw

规则：
- 四个标签页必须分函数处理
- 能结构化的表格、统计项、列表项必须结构化
- 缺失字段允许为空，不得跳过整个实体
- 必须保留来源 URL 与抓取时间

---

## XZ 职业采集要求

### Stage 1：列表页
必须提取：
- 职业名称
- careerId 或职业主键
- 详情页
- 列表来源页
- 页码

规则：
- 若页面主键未稳定确认，可先保留详情页 URL 作为临时唯一键
- 所有链接转绝对 URL
- 支持分页与筛选条件记录

### Stage 2：详情页
必须至少解析：
- 顶部信息
- 职业介绍
- 职业要求
- 就业方向
- 相关专业
- raw

规则：
- 当前以可稳定识别字段为主
- 若样例不足，先输出保守结构，不编造字段
- 页面结构确认后再细化数据结构

---

## 通用约束
- 不允许在 Stage 1 做 Stage 2 的解析工作
- 不允许省略来源 URL
- 不允许只保存 raw_text 而不做结构化提取
- 必须支持 partial 输出，避免整批失败
- 必须兼容 GitHub Actions 执行环境
